    概览

    Android Things，通过提供相同的Android开发工具, 一流的Android框架和Google API，使开发人员在移动设备上取得成功，使开发连接嵌入式设备变得容易.

    嵌入式设备的应用程序使开发人员更接近硬件外设和驱动程序，而不是手机和平板. 此外, 嵌入式设备通常向用户呈现单一的用户体验. 本文档介绍了Android开发和Android Things之间的主要补充，遗漏和差异.

    Android Things 通过物件支持库提供的附加API来扩展核心的Android框架.这些APIs允许应用程序与在移动设备上找不到的新类型的硬件集成.

    Android Things平台也简化了单一应用程序使用. 系统应用程序不存在，您的应用程序会在启动时会自动启动，让您的用户沉浸在程序体验当中.

    Things 支持库

    外部设备 I/O API

    外部设备 I/O APIs 让您的应用程序使用行业标准协议和接口，传感器，执行器通信. 支持以下接口:GPIO, PWM, I2C, SPI, UART.

    有关如何使用APIs的更多信息，请参考 Peripheral I/O API Guides.

    用户驱动 API

    用户驱动程序扩展现有的Android框架服务，并允许应用程序将硬件事件注入其他应用程序，可以使用标准Android APIs访问的框架.

    有关如何使用APIs的更多信息，请参考 User Driver API Guides.

    行为变化

    核心应用程序包

    Android Things 不包括系统应用程序和内容提供商的标准套件.避免在应用程序中使用 common intents 以及以下内容，提供者APIs:

    日历合同
    联系人合同
    文件合同
    下载管理器
    媒体商店
    设置
    电话
    用户词典
    语音信箱

    显示是可选的

    Android Things 支持使用与传统Android应用程序相同的 UI 工具包 的图形用户界面. 在图形模式下，应用程序窗口占据显示器的完整空间.Android Things 不包括系统状态栏或导航按钮，使应用程序可以完全控制视觉的用户体验.

    但是, Android Things 不需要 显示. 在不存在图形显示的设备上, 动态仍然是您的Android Things应用程序的主要组成部分. 这是因为框架将所有 输入事件 提供给具有焦点的前台活动. 您的应用程序无法通过任何其他应用组件（ 服务接收关键事件或运动事件.

    家庭活动支持

    Android Things 预期一个应用程序将其清单中的“家庭活动”公开为系统在启动时自动启动的主要入口点. 此活动必须包含一个 CATEGORY_DEFAULT 和 IOT_LAUNCHER.

    为了方便开发, 同样的活动应包含一个 CATEGORY_LAUNCHER 意图过滤器，因为 Android Studio 可以部署或调试时将其作为默认活动启动.

    <application
    android:label="@string/app_name">
    <activity android:name=".HomeActivity">
        <!-- Launch activity as default from Android Studio -->
        <intent-filter>
            <action android:name="android.intent.action.MAIN"/>
            <category android:name="android.intent.category.LAUNCHER"/>
        </intent-filter>

        <!-- Launch activity automatically on boot -->
        <intent-filter>
            <action android:name="android.intent.action.MAIN"/>
            <category android:name="android.intent.category.IOT_LAUNCHER"/>
            <category android:name="android.intent.category.DEFAULT"/>
        </intent-filter>
    </activity>
</application>

    Android Things 每个版本 都会捆绑最新版本的 Google Play 服务, 并且至少需要 10.0.0 版的客户端 SDK. Android Things 不包括 Google Play 商店, 该商店负责在设备上自动更新Play服务. 由于设备上的Play服务版本是静态的, 因此应用程序无法将客户端SDK定位到大于目标版本的版本并捆绑在一起..

    权限

    不支持在运行时请求的权限 因为嵌入式设备不能保证有UI来接受运行时对话框.在应用程序的清单文件中 声明所需的权限. 在应用程序的清单中声明的所有正常和危险的权限都将在安装时被授予.

    通知

    由于Android Thing中没有全系统状态栏和windows shade,因此不支持 通知 . 避免在您的应用程序中调用 NotificationManager APIs.

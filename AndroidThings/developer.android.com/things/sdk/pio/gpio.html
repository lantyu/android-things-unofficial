    GPIO

    通用输入/输出 (GPIO)提供了可编程接口来阅读二进制输入设备的状态（如按钮）或控制二进制输出设备的开关状态（如LED）

    你可以为GPIO配置输入和输出装态为高或低。作为外接输入决定了当前状态，以及你App能够读取的当前值或控制它做出响应。作为输出你的app可以配置pin口的状态。

    备注: 为了避免损坏GPIO pin口，请在连线前检查你的硬件的输入和输出上限。 具体请参阅 硬件 101 和你的硬件说明文档。

    管理连接

    为了打开一个GPIO端口的连接，你需要知道唯一的端口名。在开发的初始阶段，或当部署一个app到新的硬件上时，通过调用 PeripheralManagerService 的 getGpioList()来发现所有可用的端口名是非常有用的:

    PeripheralManagerService manager = new PeripheralManagerService();
List<String> portList = manager.getGpioList();
if (portList.isEmpty()) {
    Log.i(TAG, "No GPIO port available on this device.");
} else {
    Log.i(TAG, "List of available ports: " + portList);
}

    一旦你知道了目标的名字，使用 PeripheralManagerService来连接那个端口。 当你完成了和GPIO端口的通信，请关闭连接来释放资源。此外，你直到已经存在的连接关闭了，才可以开一个新的连接到同一个端口。 关闭连接请使用close()方法。 

    public class HomeActivity extends Activity {
    // GPIO Pin Name
    private static final String GPIO_NAME = ...;

    private Gpio mGpio;

    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        // Attempt to access the GPIO
        try {
            PeripheralManagerService manager = new PeripheralManagerService();
            mGpio = manager.openGpio(GPIO_NAME);
        } catch (IOException e) {
             Log.w(TAG, "Unable to access GPIO", e);
        }
    }

    @Override
    protected void onDestroy() {
        super.onDestroy();

        if (mGpio != null) {
            try {
                mGpio.close();
                mGpio = null;
            } catch (IOException e) {
                Log.w(TAG, "Unable to close GPIO", e);
            }
        }
    }
}

    读取输入

    为了能够将GPIO端口作为一个输入进行读取：

    1. 请使用 setDirection()方法，并使用 DIRECTION_IN模式，将它配置为一个输入.
    2. 配置一个高（接近IOREF）或低（接近0）压信号来作为 true （激活）值进行返回。需要调用 setActiveType() 同时使用 ACTIVE_HIGH 或 ACTIVE_LOW.
    3. 获取当前状态值请使用 getValue()方法。

    以下代码展示了，如何使用关联了高压的活动态来设置一个输入： 

    public void configureInput(Gpio gpio) throws IOException {
    // Initialize the pin as an input
    gpio.setDirection(Gpio.DIRECTION_IN);
    // High voltage is considered active
    gpio.setActiveType(Gpio.ACTIVE_HIGH);

    ...

    // Read the active high pin state
    if (gpio.getValue()) {
        // Pin is HIGH
    } else {
        // Pin is LOW
    }
}

    监听输入状态变化

    被配置为输入的GPIO端口可以在它的状态在高低值变化时通知你的app。为这些变化事件注册:

    1. 添加一个 GpioCallback 来激活端口连接。

    2. 使用 setEdgeTriggerType()方法可以声明当状态变化时触发一个中断事件。 这个边缘触发器支持以下四种类型:

      • EDGE_NONE: 无中断事件。 这是默认值。
      • EDGE_RISING: 中断从低信号到高信号的变化。
      • EDGE_FALLING: 中断从高信号到低信号的变化。
      • EDGE_BOTH: 中断所有状态变化。

    3. onGpioEdge()返回true 表明侦听器应该继续接受每个端口状态变化的事件。

      以下的code注册了一个侦听所有输入端口状态变化的中断侦听器： 

      public void configureInput(Gpio gpio) throws IOException {
    // Initialize the pin as an input
    gpio.setDirection(Gpio.DIRECTION_IN);
    // Low voltage is considered active
    gpio.setActiveType(Gpio.ACTIVE_LOW);

    // Register for all state changes
    gpio.setEdgeTriggerType(Gpio.EDGE_BOTH);
    gpio.registerGpioCallback(mGpioCallback);
}

private GpioCallback mGpioCallback = new GpioCallback() {
    @Override
    public boolean onGpioEdge(Gpio gpio) {
        // Read the active low pin state
        if (mDevice.getValue()) {
            // Pin is LOW
        } else {
            // Pin is HIGH
        }

        // Continue listening for more interrupts
        return true;
    }

    @Override
    public void onGpioError(Gpio gpio, int error) {
        Log.w(TAG, gpio + ": Error event " + error);
    }
};

    4. 当你的app不再侦听新来的事件时，注销所有的终端处理： 

      public class HomeActivity extends Activity {
    private Gpio mGpio;
    ...

    @Override
    protected void onStart() {
        super.onStart();

        // Begin listening for interrupt events
        mGpio.registerGpioCallback(mGpioCallback);
    }

    @Override
    protected void onStop() {
        super.onStop();
        // Interrupt events no longer necessary
        mGpio.unregisterGpioCallback(mGpioCallback);
    }
}

    向输出写数据

    用程序控制GPIO端口的状态:

    1. 使用 setDirection() 将其配置为输出口，并使用 DIRECTION_OUT_INITIALLY_HIGH 或 DIRECTION_OUT_INITIALLY_LOW模式. 这些模式保证了端口状态在配置时被正确的初始化。
    2. 配置一个高（接近IOREF）或低（接近0）压信号来作为 true （激活）值进行返回。需要调用 setActiveType() 同时使用 ACTIVE_HIGH 或 ACTIVE_LOW.
    3. 使用setValue() 方法设置当前状态.

    以下代码展示了如何设置输出口，并初始化为信号高值，然后使用 setValue() 方法切换其状态为信号低值:

    public void configureOutput(Gpio gpio) throws IOException {
    // Initialize the pin as a high output
    gpio.setDirection(Gpio.DIRECTION_OUT_INITIALLY_HIGH);
    // Low voltage is considered active
    gpio.setActiveType(Gpio.ACTIVE_LOW);

    ...

    // Toggle the value to be LOW
    gpio.setValue(true);
}
